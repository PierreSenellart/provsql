# Building ProvSQL for the browser (WebAssembly / PGlite)

This directory builds ProvSQL as a WebAssembly extension that loads into
**[PGlite](https://pglite.dev/)** (PostgreSQL compiled to WASM), so the
whole stack – provenance rewriting, the circuit store, semirings, and
probability – runs **client-side in a browser**, with no server.

See [`doc/source/dev/playground.rst`](../doc/source/dev/playground.rst)
for the design. The extension itself builds from the normal
`src/` tree under the `PROVSQL_INPROCESS_STORE` / `PROVSQL_NO_SUBPROCESS`
flags (auto-defined under `__EMSCRIPTEN__`): a single-process in-memory
circuit store, no background worker, no `fork`/`exec`/sockets, and no
compiled `libboost_serialization`.

## What you need

- **podman** (rootless; no daemon/root needed) or Docker, to run the
  pinned builder image `electricsql/pglite-builder:3.1.74-5-postgis-libicu-min`
  (Emscripten 3.1.74 + WASM-built deps).
- **Node 20+** and **pnpm** (via `corepack`) to build the PGlite TS package.
- Host **Boost headers** (`/usr/include/boost`; header-only – ProvSQL needs
  no compiled Boost in this build).

## Recipe

`make wasm` (from the repo root) runs this whole recipe via
[`build-wasm.sh`](build-wasm.sh) -- it builds the artifacts and assembles the
Playground doc-root -- so the manual steps below are mainly for reference and
debugging. The browser e2e is then `make playground-test`.

```bash
ROOT=$PWD                      # the provsql repo
IMG=docker.io/electricsql/pglite-builder:3.1.74-5-postgis-libicu-min
podman pull "$IMG"

# 1. PGlite's patched Postgres tree (the WASM build host).
git clone --recurse-submodules https://github.com/electric-sql/pglite
cd pglite && git submodule update --init --depth 1 postgres-pglite
PG=$PWD/postgres-pglite

# 2. Build the WASM Postgres core once (upstream configure+make+install;
#    the `dist/` install is what the extension links against). This is the
#    first ~two steps of postgres-pglite/build-pglite.sh, run in the image.
podman run --rm --workdir="$PG" -v "$PG:$PG:rw" -v "$PG/dist:/pglite:rw" "$IMG" ./build-pglite.sh
#    (or run only its configure + `emmake make` + `emmake make install`.)

# 3. Stage ProvSQL's sources + generated SQL into the extension build dir.
mkdir -p "$PG/provsql-wasm"
cp -r "$ROOT/src" "$PG/provsql-wasm/src"
cp "$ROOT/sql/provsql--"*.sql "$PG/provsql-wasm/"     # generated by `make` in $ROOT
cp "$ROOT/provsql.control" "$PG/provsql-wasm/"
# `relink-pglite.sh` is deliberately a different name from the upstream
# `build-pglite.sh` (step 2) so it does not clobber the core builder.
cp "$ROOT/wasm/build-extension.sh" "$ROOT/wasm/relink-pglite.sh" "$ROOT/wasm/test-node.mjs" "$PG/"

# 4. Compile + package the extension, then relink a matched pglite.wasm
#    that exports the symbols ProvSQL imports.
podman run --rm --workdir="$PG" -v "$PG:$PG:rw" -v "$PG/dist:/pglite:rw" \
  -v /usr/include/boost:/boostinc/boost:ro "$IMG" ./build-extension.sh
podman run --rm --workdir="$PG" -v "$PG:$PG:rw" -v "$PG/dist:/pglite:rw" "$IMG" ./relink-pglite.sh

# 5. Build the PGlite TS package against the matched wasm, bundling the
#    contrib tarballs (uuid-ossp, ...) ProvSQL's CASCADE needs.
cp "$PG"/dist/bin/pglite.{wasm,data,js} packages/pglite/release/
# Contrib tarballs (uuid-ossp…) that `CREATE EXTENSION provsql CASCADE`
# needs: from the full build's dist/extensions, or – if you only built the
# core – copy them from the published npm package (ABI-compatible, same
# tree): `cp node_modules/@electric-sql/pglite/dist/*.tar.gz packages/pglite/release/`
corepack pnpm install --filter "@electric-sql/pglite..."
corepack pnpm --filter @electric-sql/pglite-utils run build
corepack pnpm --filter @electric-sql/pglite      run build:js

# 6. Headless smoke test (asserts correct provenance + probability).
cp "$ROOT/wasm/test-node.mjs" .
PGLITE_DIST=$PWD/packages/pglite/dist \
PROVSQL_TARBALL=$PG/provsql-wasm/provsql.tar.gz \
  node test-node.mjs
```

The artifacts are the **matched pair**: `packages/pglite/dist/pglite.wasm`
(+ `.data`/`.js`) and `provsql.tar.gz`. Load them in a browser by serving
`packages/pglite/dist` + the tarball and wiring a tiny `Extension` whose
`bundlePath` points at `provsql.tar.gz` (see `studio/web/`).

## Notes

- **`MAIN_MODULE=2`, not `1`.** Export-all (`MAIN_MODULE=1`) links but
  breaks `initdb`; the matched build keeps `MAIN_MODULE=2` and adds
  ProvSQL's imports to the export list (step 4 / `relink-pglite.sh`).
- **The libc++ `inline` fix** (`patches/0001-cxx-inline-libcxx.patch`) is a
  PGlite-Postgres bug that blocks any C++ extension; ideally upstreamed to
  `electric-sql/postgres-pglite`.
- Loading the compiled `provsql.so` into *stock npm* PGlite is **not**
  possible – its export set is too narrow. A matched build is required.
- PGlite's extension API is not yet stable; this targets PGlite 0.4.x /
  PostgreSQL 17.5, built from the `postgres-pglite` commit pinned by the
  PGlite release used.
